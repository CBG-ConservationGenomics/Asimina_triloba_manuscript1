# Landscape Genomics Pipeline — *Asimina triloba* (pawpaw)

An R-based landscape genomics workflow for eastern North American pawpaw. The pipeline links genotype, geography, and climate data through diversity mapping, population structure, isolation-by-distance/environment (IBD/IBE), genotype–environment association (GEA), linkage decay, GO enrichment, and edge-vs-core allele analysis.

Analysis methods are implemented primarily with [**algatr**](https://thewanglab.github.io/algatr/) (Wang Lab), with auxiliary tools including **wingen**, **PLINK**, **bedtools**, and **topGO**.

## Pipeline overview

Scripts are numbered and intended to run in dependency order. Steps **01**, **02**, and **03** can run in parallel after **00** completes. Step **04** should follow **00** and **03** (for MMRR summary panels). Steps **05 → 06** form a secondary branch from GEA results. Step **07** is an optional extension for range-margin allele analysis.

### High-level workflow

| Phase | Scripts | Question addressed |
|-------|---------|-------------------|
| **Prepare** | 00 | Load VCF, LD-prune, impute, extract/scale climate, build shared RDS objects |
| **Describe** | 01–02 | Where is genetic diversity? How is population structure organized spatially? |
| **Associate** | 03–04 | Is genetic distance explained by geography and environment (MMRR/GDM)? Which SNPs associate with climate (RDA/LFMM)? |
| **Mechanism** | 05–06 | Which genes are near GEA SNPs? Are they enriched for GO terms? |
| **Range margin** | 07 | Which alleles are private or enriched at edge vs core populations? |

## Script reference

### Shared: `lgp_pipeline_cache.R`

Utility functions sourced by scripts **00–07**:

- **`lgp_project_root()`** — resolves project directory (`LGP_PROJECT_ROOT` or `options(lgp.project_root)`).
- **`lgp_preprocess_dir()`** / **`lgp_outputs_step_dir()`** — standardized paths to `Outputs/00-preprocessing/` and per-step folders.
- **`lgp_rds_cached()`**, **`lgp_list_cached()`** — skip expensive blocks when cache is newer than inputs.
- **`lgp_pack_for_rds()`** / **`lgp_read_rds()`** — safe serialization of `terra` rasters (avoids stale external pointers).

See `Scripts/PIPELINE_CACHE.md` for force-rerun flags and troubleshooting.

### 00 — Preprocessing

**Script:** `Scripts/00-Preprocessing.R`

**Purpose:** Build all shared objects used downstream: pruned VCF, imputed dosage matrix, genetic distances, and scaled environmental predictors at sample locations.

**Main steps:**

1. Load `InputData/sampling_coords.tsv` and WorldClim bioclim layers (0.5°) at sample points.
2. LD-prune the landscape VCF (`ld_prune`, r² = 0.9, window 10) if the pruned file is older than the source VCF.
3. Convert VCF → dosage matrix; impute missing genotypes with **`str_impute()`** (K = 1–6, 5 repetitions).
4. Compute Euclidean **`gen_dist()`** on imputed dosages.
5. Collinearity QC on bioclim variables; retain subset: `bio2`, `bio7`, `bio8`, `bio9`, `bio13`, `bio15`, `bio18`.
6. Extract raw and **z-scored** environmental values per sample.
7. Build **`snp_map`** (SNP name, chromosome, position) for Manhattan plots later.

**Key outputs** (`Outputs/00-preprocessing/`):

| File | Description |
|------|-------------|
| `coord.rds` | Sample coordinates |
| `str_dos.rds` | Imputed dosage matrix (samples × SNPs) |
| `vcf_pruned.rds` | LD-pruned VCF object |
| `euc_gendist.rds` | Pairwise Euclidean genetic distance |
| `env_scaled.rds` | Scaled environmental matrix |
| `selected_wclim.rds` | Subset WorldClim raster stack |
| `snp_map.rds` | SNP ↔ chr/pos lookup |
| `bio_colinearity.tsv`, `mantel_test.txt` | Environmental QC tables |

**Prerequisites:** `InputData/filtered_snps4landscape_reheader.vcf`, `sampling_coords.tsv`.

### 01 — Genetic diversity

**Script:** `Scripts/01-GeneticDiversity.R`

**Purpose:** Map nucleotide diversity (π) and sample density across the study region using **wingen** moving windows, with species range and Eastern Continental Divide as map context.

**Main steps:**

1. Project samples and rasters to a local UTM CRS.
2. Run wingen moving-window analyses (π, sample counts).
3. Optional ordinary-kriging surfaces for visualization.
4. Overlay **Little (1977)** pawpaw range (`InputData/Pawpaw_range_Little_1977.zip`) as grey underlay on maps.
5. Export manuscript-ready π summary statistics.

**Key outputs** (`Outputs/01-genetic-diversity/`):

| File | Description |
|------|-------------|
| `nucleotide_diversity_summary.tsv` | **MIN_PI**, **MAX_PI**, **MEAN_PI** (and related stats) |
| `nucleotide_diversity_cells_min2.tsv` | Per-cell π where ≥ 2 samples |
| `wingen_moving_window_pi.png` | π surface map |
| `wingen_kriged_*.png` | Kriged diversity / sample-count maps |
| `projected_sample_locations_*.png` | Sample map in projected CRS |

**Prerequisites:** Run **00**; `Pawpaw_range_Little_1977.zip` for range underlay.

**Reference:** [wingen / algatr diversity vignette](https://thewanglab.github.io/algatr/articles/wingen_vignette.html)

### 02 — Population structure

**Script:** `Scripts/02-PopulationStructure.R`

**Purpose:** Infer spatial population structure with **TESS3** and relate ancestry clusters to geography and environment.

**Main steps:**

1. Run TESS3 on imputed dosages with sample coordinates.
2. Export Q-matrix per individual and per inferred cluster (K).
3. Map cluster membership; overlay species range.
4. Hierarchical clustering dendrogram + Newick export.
5. Correlate environmental variables with ancestry (`tess_qmatrix.tsv` for downstream use in **07**).

**Key outputs** (`Outputs/02-population-structure/`):

| File | Description |
|------|-------------|
| `tess_qmatrix.tsv` / `.rds` | Individual ancestry coefficients |
| `hclust_average.newick` | Clustering tree |
| `*.png` | TESS maps, bar plots, dendrograms |

**Prerequisites:** Run **00**.

**Reference:** [TESS vignette](https://thewanglab.github.io/algatr/articles/TESS_vignette.html)

### 03 — IBD / IBE

**Script:** `Scripts/03-IBD-IBE.R`

**Purpose:** Partition genetic variation into geographic vs environmental components using **MMRR** (individual-level) and **GDM** (population-level).

**Main steps:**

1. **MMRR** — regress pairwise genetic distance on environmental distance matrices + geographic distance (`geodist`); full and best-subset models; 999 permutations.
2. **GDM** — population-level generalized dissimilarity modeling with allele-frequency filtering; variable importance and response curves.
3. Cache MMRR fit objects for reuse in **04** (coefficient panels).

**Key outputs** (`Outputs/03-ibd-ibe/`):

| File | Description |
|------|-------------|
| `results/mmrr_results_full.txt` | Standardized MMRR coefficients and p-values |
| `plots/mmrr_*.png` | MMRR diagnostic and variable plots |
| `results/gdm_table_full.txt`, `gdm_varimp_table.txt` | GDM fit and variable importance |
| `_step_cache/mmrr_YX_models.rds` | Cached MMRR objects (read by **04**) |

**Prerequisites:** Run **00** (`euc_gendist.rds`, `env_scaled.rds`, `coord.rds`).

**References:** [MMRR](https://thewanglab.github.io/algatr/articles/MMRR_vignette.html), [GDM](https://thewanglab.github.io/algatr/articles/GDM_vignette.html)

### 04 — GEA (RDA & LFMM)

**Script:** `Scripts/04-GEA.R`

**Purpose:** Identify SNPs associated with environment via **redundancy analysis (RDA)** and **latent factor mixed models (LFMM)**.

**Main steps:**

1. Hellinger-transform genotypes; run full and best-subset RDA; partial RDA controlling for geography and genetic structure (PCs).
2. Variance partitioning among environment, geography, and structure.
3. Outlier SNPs via z-scores and FDR-adjusted p-values (`rda_getoutliers`).
4. LFMM ridge regression (K = 2); genome-wide Manhattan and QQ plots.
5. Export significant SNP lists for **05** and **07**.
6. **MMRR panels** — coefficient forest plot and relative-importance bar chart using live results from **03** (not hardcoded).

**Key outputs** (`Outputs/04-gea/`):

| File | Description |
|------|-------------|
| `results/rda_p0.05.csv` | RDA SNP–environment correlations (p < 0.05) |
| `results/best_pRDA_sigZ_SNPS.csv` | Z-score RDA outliers |
| `results/sign_lfmm_ridge_table.csv` | LFMM ridge significant hits |
| `results/lfmm_ridge_results.csv` | Full LFMM results |
| `results/mmrr_coefficient_table.tsv` | MMRR table used for panels |
| `plots/pRDA_z_manhattan_chr.png` | RDA genome scan by chromosome |
| `plots/lfmm_manhattan_ridge.png` | LFMM Manhattan |
| `plots/mmrr_panel_coefficients.png` | MMRR standardized coefficients |
| `plots/mmrr_panel_importance.png` | MMRR relative importance |

**Prerequisites:** Run **00**; **03** recommended before MMRR panels.

**Reference:** [RDA vignette](https://thewanglab.github.io/algatr/articles/RDA_vignette.html)

### 05 — Linkage decay

**Script:** `Scripts/05-LinkageDecay.R`

**Purpose:** Characterize LD around GEA candidate SNPs and link SNPs to annotated genes.

**Main steps:**

1. Run **PLINK** genome-wide LD on the landscape VCF (cached if outputs are fresh).
2. Read GEA SNP lists from **04** (`rda_p0.05.csv`, `sign_lfmm_ridge_table.csv`).
3. Focal LD decay plots around top SNPs per analysis.
4. **bedtools** window intersection with `a_triloba_filtered_longest_isoform_gene.gff` → linked gene tables.

**Key outputs** (`Outputs/05-linkage-decay/`):

| File | Description |
|------|-------------|
| `pawpaw_LD.ld.gz` | Genome-wide PLINK LD table |
| `results/pRDA_linked_genes.tsv` | Genes near RDA SNPs |
| `results/LFMM_linked_genes.tsv` | Genes near LFMM SNPs |
| `plots/LD_top_*.png` | Focal LD decay curves |

**Prerequisites:** Run **04**; **PLINK** and **bedtools** on `PATH`; GFF in `InputData/`.

### 06 — GO enrichment

**Script:** `Scripts/06-GO_enrichment.R`

**Purpose:** Test whether genes linked to GEA SNPs are enriched for Gene Ontology terms (MF, BP, CC) using **topGO**.

**Main steps:**

1. Load linked-gene tables from **05**.
2. Map genes to GO terms (`gene_go_terms_wide.tsv`) and descriptions.
3. Run Fisher tests per ontology; export term lists and optional heatmap matrix.

**Key outputs** (`Outputs/06-go-enrichment/`):

| File | Description |
|------|-------------|
| `results/pRDA_*_go_terms.csv` | GO enrichment for RDA-linked genes |
| `results/LFMM_*_go_terms.csv` | GO enrichment for LFMM-linked genes |
| `results/*_linked_unique_genes.csv` | Deduplicated gene sets tested |
| `plots/go_enrichment_heatmap_matrix.csv` | Heatmap input matrix |

**Prerequisites:** Run **05**; Bioconductor **topGO** / **GO.db**; GO annotation files in `InputData/`.

### 07 — Edge vs core alleles

**Script:** `Scripts/07-EdgeCoreAlleles.R`

**Purpose:** Landscape-genomics “part 2” — per-population allele frequencies, private/near-private alleles, edge-vs-core Fst, candidate SNP tables, and optional overlap with GEA hits.

**Main steps:**

1. Assign populations to **edge** vs **core** (default: distance to Little 1977 range margin; also supports manual lists, latitude margin, TESS, ECD).
2. Compute per-population allele frequencies.
3. Flag **private / near-private** alleles per edge population (high in one edge site, rare elsewhere).
4. Weir–Cockerham Fst between pooled edge and core; Fst outlier threshold.
5. Candidate SNP table (high Fst, private, fixation patterns); optional intersection with **04** GEA lists.
6. Maps: edge/core assignment, GEA candidate frequency maps with species-range underlay.

**Key outputs** (`Outputs/07-edge-core-alleles/`):

| File | Description |
|------|-------------|
| `results/population_edge_core_assignment.tsv` | Population labels + optional dist-to-range |
| `results/pop_allele_frequencies_long.tsv` | Long-format allele frequencies |
| `results/private_per_edge_population.tsv` | SNP × edge pop private flags |
| `results/private_and_near_private_snps.tsv` | SNP-level private summary |
| `results/snp_fst_edge_core.tsv` | Per-SNP edge–core Fst |
| `results/edge_core_snp_candidates.tsv` | Combined candidate flags |
| `results/edge_core_candidates_x_gea.tsv` | Candidates overlapping GEA |
| `plots/population_edge_core_*.png` | Edge/core map |
| `plots/gea_candidate_freq_maps/` | Per-SNP frequency maps |

**Configuration:** See `InputData/README_edge_core.txt` for `options(lgp.edge_core.*)` (method, private thresholds, manual population lists).

**Prerequisites:** Run **00**; **04** optional (GEA overlap and frequency maps).

**Map-only rerun:**

```bash
LGP_07_MAP_ONLY=1 Rscript Scripts/07-EdgeCoreAlleles.R
```

## Incremental reruns & caching

The pipeline caches expensive steps to speed up iterative work:

```bash
# Force rebuild all RDS caches
export LGP_FORCE_RERUN=1

# Force external tools (PLINK) even if outputs look fresh
export LGP_FORCE_EXTERNAL=1
```

In R:

```r
options(lgp.force_rerun = TRUE)
options(lgp.gea_force_lfmm = TRUE)   # always refit LFMM in 04
```

Details: `Scripts/PIPELINE_CACHE.md`.

## Configuration

| Setting | How to set | Effect |
|---------|------------|--------|
| Project root | `export LGP_PROJECT_ROOT=/path/to/repo` | All scripts resolve `InputData/` and `Outputs/` |
| | `options(lgp.project_root = "...")` | Same, from R |
| Edge/core method | `options(lgp.edge_core.method = "species_range")` | See `InputData/README_edge_core.txt` |
| Private allele rules | `options(lgp.edge_core.private_scope = "per_edge_pop")` | Per edge population vs pooled |
| LFMM threshold | `options(lgp.lfmm_adj_p_threshold = 0.05)` | Significance cutoff in **04** |

Population IDs in **07** are **site-level** names derived from sample IDs (text before the first `-`; `WC_PA_*` → `WC_PA`), consistent with **03** GDM.
